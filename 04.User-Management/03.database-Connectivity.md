## Database Connectivity: 

Database connectivity is how a client application (SQL*Plus, Java, Python, etc.) connects to an Oracle database to send queries and receive results.

It involves:
- **Network layer**: TCP/IP, port (default `1521`)
- **Listener**: Oracle service that accepts client connections
- **Service/SID**: The target database
- **Authentication**: Username/password or OS authentication


### Types of Oracle Connections:

#### 1. Local Connection:
- Client is on the same server as the database.
- Can use OS authentication, no password needed.


#### 2. Remote Connection:
- Client is on a different machine.
- Uses TNS (Oracle Net) or Easy Connect.




### Connection Flow:

```
Client (SQL*Plus, Java, Python)
        │
        │ TCP/IP
        ▼
Oracle Listener (1521)
        │
        ▼
Database Service (non-CDB, CDB or PDB)
        │
        ▼
Authenticate user (SYS, APPUSER, etc.)
        │
        ▼
Access schema/tables
```

## Connect with Username & Password: 

### Connect to non-CDB and CDB (`sys` user): (TNS Connection)

- TNS Connection (`tnsnames.ora`): This is the traditional Oracle connection method.
- Here `orcl` and `orcldb` is defined inside the `tnsnames.ora` file. (check: `tnsping orcl` and `tnsping orcldb`)
- Disadvantage: Requires configuration file



```
sqlplus sys/<password>@<service_name> as sysdba

sqlplus sys/oracle@orcl as sysdba
sqlplus sys/oracle@orcldb as sysdba
```



### Connect to non-CDB and CDB (`sys` user): Easy Connect (EZCONNECT)
- No need `tnsnames.ora`
- Disadvantage: Hard to manage many connections

```
sqlplus sys/<password>@server_IP:port/<database> as sysdba

sqlplus sys/oracle@10.1.1.10:1521/orcl as sysdba
sqlplus sys/oracle@10.1.1.11:1521/orcldb as sysdba
```

## Connect from Oracle SQL Developer:

### Connect to non-CDB and CDB (`sys` user):

1. New Connection:
    - Connection Name: 
    - Username: `sys`
    - Password: `sys_password`
    - Connection Type: `Basic` -> Role: `SYSDBA`
    - Hostname: `10.1.1.10`
    - Port: `1521`
    - Service name: `orcl` or `orcldb`
    - Click: Test (Status: Success)
    - Click: Connect


2. New Connection:
    - Connection Name: 
    - Username: `system`
    - Password: `system_password`
    - Connection Type: `Basic` -> Role: `default`
    - Hostname: `10.1.1.10`
    - Port: `1521`
    - Service name: `orcl` or `orcldb`
    - Click: Test (Status: Success)
    - Click: Connect



### Connect to non-CDB and CDB (local user):

1. New Connection:
    - Connection Name: 
    - Username: `appuser` or `c##myadmin`
    - Password: `app123`
    - Connection Type: `Basic` -> Role: `default`
    - Hostname: `10.1.1.10`
    - Port: `1521`
    - Service name: `orcl` or `orcldb`
    - Click: Test (Status: Success)
    - Click: Connect


### Connect to PDB (local user):

1. New Connection:
    - Connection Name: 
    - Username: `appuser` 
    - Password: `app123`
    - Connection Type: `Basic` -> Role: `default`
    - Hostname: `10.1.1.11`
    - Port: `1521`
    - Service name: `LOCALPDB1` 
    - Click: Test (Status: Success)
    - Click: Connect




### Summary:

- **Local connection**: OS authentication (sqlplus / as sysdba)
- **Remote connection**: TNS or Easy Connect
- **CDB/PDB**: Use local users for PDBs, common users for CDB & PDBs
- **Key DBA checks**: service name, SID, DB_NAME, DB_UNIQUE_NAME, listener status
